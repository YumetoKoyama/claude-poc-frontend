# TODO コメントルール

## 基本方針

後続タスクに課題を残す場合または、後続タスクで実装することが前提の内容を実装する場合は、以下のルールを遵守すること。

## TODO コメントに含める情報

TODO コメントを使用して、後続タスクで実装する内容を明確に示すこと。各コメントには以下を含めること。

- 後続タスクで実装する内容の概要
- 後続タスクで実装する内容の詳細な説明
- 後続タスクで実装する内容に関連する情報（例：関連するファイルやコードの場所、関連するドキュメントなど）
- 暫定的に実装した内容がある場合は、その内容と理由を明記すること

## 運用ルール

- TODO コメントは、実装する内容が明確に理解できるように、具体的かつ詳細に記述すること
- TODO コメントは、実装する内容が完了したら、必ず削除すること
- TODO コメントは、コードの可読性を損なわないように、適切な場所に記述すること
